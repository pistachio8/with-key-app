### 2-2. 재발 off-vocab 요청 클러스터 — 추가 후보 키워드

**모두 `confidence=0.2, scores={}`로 analysis 폴백. 사람이 clarify 게이트에서 수정하거나 의도와 다른 워크플로로 진행된 요청들.**

#### 클러스터 A — "구현" 단독 (1건)

- 원문: `"EVAL-0043 영상 캡처·저장 + 스토리 자동재생(Phase 1) 구현"`
- 의도 타입: `feature`
- 미매칭 이유: `feature` 키워드 표에 `"구현해줘"` 는 있으나 `"구현"` 단독은 없음. 사용자가 EVAL ID + 기술 명사 + 동사 형태로 요청할 때 `"해줘"` 를 생략하는 자연스러운 형태.
- **추가 후보**: `"구현"` → `feature` CLASSIFIER_KEYWORDS
- 주의: `"구현"` 은 일반 명사로도 쓰여 오탐 가능성 있음. `"구현해"` 로 좁히는 안도 검토 권장.

#### 클러스터 B — "WP 분해" / "task 분해" (1건)

- 원문: `"RN Phase 6 알림(Expo push token 등록·notification prefs·deadline/friend/kudos/start dispatch·local notification store) WP/Agent Task 분해 — create-agent-tasks"`
- 의도 타입: `feature` (feature 계획 단계 — WP 분해 → Agent Task 생성)
- 미매칭 이유: `"WP 분해"`, `"task 분해"`, `"분해"` 가 어떤 타입 표에도 없음. `"알림"` 은 도메인 키워드(push)로만 잡히고 타입은 결정 못 함.
- **추가 후보**: `"분해해"` → `feature` CLASSIFIER_KEYWORDS
  - `"분해"` 단독은 광범위. `"분해해"` 또는 `"WP 분해"` 로 좁히는 것 권장. harness-improvement 우선 키워드와 겹치지 않는 범위에서 결정 필요.

#### 클러스터 C — "ADR 작성" / "초안 작성" (1건)

- 원문: `"D-2 push token 모델 ADR 초안 작성"`
- 의도 타입: `docs` 또는 `prd` (ADR 문서 작성)
- 미매칭 이유: `"adr"` 이 어떤 타입 표에도 없음. `docs` 키워드 표는 `"문서화"`, `"readme"`, `"문서 정리"`, `"주석 추가"`, `"가이드 작성"` 만 보유. `prd` 표도 `"prd"`, `"기획서"`, `"요구사항"` 등 — `"adr"` 없음.
- **추가 후보**: `"adr"` → `docs` CLASSIFIER_KEYWORDS (또는 `prd` — 아래 판단 필요)
  - `"초안 작성"` 추가도 검토 가능. 단 `"작성"` 단독은 광범위.
  - `adr` 이 `docs`냐 `prd`냐: ADR 은 하네스 맥락에서 되돌리기 비용 큰 결정 문서(spec-required 경로). 새 ADR 작성 요청은 `prd`(PO 승인 포함) 또는 `docs`(문서 작성) 양쪽 근거가 있음 → PO/하네스 관리자 판단 필요.

**Migration 번호 충돌**: task 파일 작성 시점과 구현 시점 사이에 다른 migration 이 먼저 머지되어 번호가 밀리는 패턴. EVAL-0044 에서 0053→0055 로 자동 조정됨(append-only 규칙 준수). 반복 시 task 파일에 "예약 번호" 기재를 폐지하거나 구현 직전 번호를 동적으로 조회하는 방향으로 개선 검토 가능.

| 순위 | 후보                                                                                      | 근거 지표                                                                   | Level                                          | 권장 라우트                                                                             |
| ---- | ----------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- | ---------------------------------------------- | --------------------------------------------------------------------------------------- |
| 1    | feature 키워드 `"구현"` (또는 `"구현하"`) 추가                                            | off-vocab 3/17 의 1건; "구현해줘" 는 있으나 단독 "구현" 미매칭              | L1                                             | propose-harness-update → harness:drift 후 apply-harness-update                          |
| 2    | feature 키워드 `"분해해"` (또는 `"WP 분해"`) 추가                                         | off-vocab 3/17 의 1건; WP/task 분해 요청이 analysis 폴백                    | L1                                             | propose-harness-update → 동상                                                           |
| 3    | docs 키워드 `"adr"` 추가 (`"초안"` 병행 검토)                                             | off-vocab 3/17 의 1건; ADR 작성 요청이 analysis 폴백                        | L1 (타입 결정이 docs냐 prd냐는 L3)             | propose-harness-update; 타입 결정은 PO 확인 선행                                        |

```bash
# run 파일 목록
ls /Users/ian/gitlab/with-key/evals/runs/ | head -20

# 라우팅 집계
node -e "
const fs = require('fs');
const runs = fs.readdirSync('evals/runs').filter(f => f.endsWith('.json'));
const data = runs.map(f => JSON.parse(fs.readFileSync('evals/runs/' + f, 'utf8')));
console.log('Total:', data.length);
const offVocab = data.filter(d => d.ambiguous && d.confidence === 0.2);
console.log('off-vocab:', offVocab.length);
const ambiguous = data.filter(d => d.ambiguous === true);
console.log('ambiguous:', ambiguous.length);
const classDistrib = {};
data.forEach(d => { classDistrib[d.classification] = (classDistrib[d.classification] || 0) + 1; });
console.log('Classification:', JSON.stringify(classDistrib));
"

# 결과 집계
node -e "
const fs = require('fs');
const data = JSON.parse(fs.readFileSync('evals/results/agent-results.json', 'utf8'));
const runs = data.runs;
console.log('Total runs:', runs.length);
const statusCounts = {};
for (const r of runs) { statusCounts[r.status] = (statusCounts[r.status] || 0) + 1; }
console.log('Status:', JSON.stringify(statusCounts));
const verdicts = {};
for (const r of runs) {
  if (r.review && typeof r.review === 'object' && r.review.verdict) {
    verdicts[r.review.verdict] = (verdicts[r.review.verdict] || 0) + 1;
  } else if (r.review && typeof r.review === 'string') {
    verdicts['self-review'] = (verdicts['self-review'] || 0) + 1;
  }
}
console.log('Verdicts:', JSON.stringify(verdicts));
"

# reason 필드 미저장 확인

const fs = require('fs');
const runs = fs.readdirSync('evals/runs').filter(f => f.endsWith('.json'));
const withReason = runs.filter(f => 'reason' in JSON.parse(fs.readFileSync('evals/runs/' + f, 'utf8')));
console.log('runs with reason field:', withReason.length, '/', runs.length);
"

# drift report 재발 패턴
grep -l 'mobile.*exit 1' /Users/ian/gitlab/with-key/evals/drift-reports/*.md
```
