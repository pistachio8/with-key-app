흐름 데이터 흐름 한눈에

화면들이 어느 단계에 놓이는지 — spec §Design "데이터 흐름 한눈에" 발췌.

[생성] 피드 타입(이미지·영상) + 벌금 + 만회 찬스 미션(자유 입력) ← 화면 1 │ [인증] 이미지 → 사진(기존) / 영상 → 실시간 3초 캡처 ← 화면 2 │ [종료/정산] 보증금 전액 환급(+H). ※ 만회 찬스가 있는 챌린지는 벌금 차감 보류(deferred) │ [창1 +0~48h] 일반 인증 peer-reject → 미달분 X 확정(동결) │ [창2 +48~96h] 미달자 → 미션 수행 녹화 영상 제출 ←→ 동료 판단 ← 화면 3·4 │ [판단] 기본=인정. 서약자 과반 '불성실' 반려 시 면제 실패 ├─ 인정 → 벌금 면제 (추가 차감 없음) └─ 반려/미제출 → 2배 빚 → 같은 그룹 다음 정산에 −2X carry-over ← 변이 D │ [결과물] 이미지 → 기존 recap / 영상 → 스토리 자동재생(Phase 1) ← 화면 5 · 합본 몽타주(fast-follow) ← 변이 E
진입점 주의 — 만회 찬스 창(창2)은 챌린지가 closed 된 뒤 열립니다. 현행 홈은 closed 를 안 보여줘서 진입로가 없습니다. 그래서 홈에 "만회 찬스" 섹션(화면 6)을 추가하고 창2 오픈 시 푸시 1회를 보냅니다.

매핑 화면 ↔ 라우트 ↔ WP

신규 화면 5개(핵심 4 + fast-follow 1) · 기존 화면 수정 2곳. 만회 찬스 라우트는 제출/판단을 분리(권장)해 2화면으로 셉니다.
