## TL;DR — 1분 요약

- **무엇**: 친구 3~4명이 **서약서 쓰고 주간 운동 챌린지**를 진행, 실패 시 벌금 · 성공 시 보상을 받는 **그룹 기반 습관 강제 앱**.
- **누구를 위해**: **"3년차 직장인 민지"** — 혼자 헬스장 등록해 3주 만에 중단을 반복하는 2030 친구 네트워크 보유자 (§2 페르소나).
- **왜 지금**: 혼자 하는 습관 앱은 이탈이 심합니다. **그룹 압박 + 원탭 키워드 기반 AI 기록**으로 지속률을 끌어올릴 수 있다는 가설.
- **이번 POC(2주)에서 검증할 단 하나의 질문**:
  > **"사람들이 운동 인증을 위해 매주 3번 이 앱을 열까?"**
- **성공 기준**: 1인당 주 평균 **2회 이상 인증** · 그룹 유지율 **50% 이상** · 서명률 **70% 이상**.
- **운동은 첫 번째 use case일 뿐** — 내부 도메인은 범용 "행동 기록"으로 설계, UI만 운동에 집중.

## 1. 비전 & 북극성

**북극성(North Star)** 은 제품이 장기적으로 나아가는 방향입니다. 단기적인 기능이 흔들릴 때 방향을 재조정하는 기준점 역할을 합니다.

- **장기 비전**: 모든 습관을 그룹 단위로 강제하는 **"행동 강제 플랫폼"**.
- **단기 목표(POC)**: 운동 챌린지 앱으로 **그룹 압박 가설**을 검증.
- **설계 원칙**: 내부는 범용(`ActionLog`/`Challenge`/`Verification`), UI는 운동에 집중 → 확장 가능성은 코드에 숨겨 둡니다.

## 2. 타겟 페르소나

레퍼런스 3건(§8 Strava · 챌린저스 · Setlog) 분석의 데이터 교집합에서 도출했습니다. "감"이 아닌 **자료 기반 페르소나**입니다.

### 2.1 도출 배경 — 3건 레퍼런스 교집합

| 앱 | 주 사용자층 | 시사점 |
|---|---|---|
| **Strava (2025)** | 20~40대 러닝/사이클 + Gen Z "운동 = 새 사회활동" (러닝 클럽 3.5배 증가) | 운동은 **지금 가장 핫한 소셜 동기** |
| **챌린저스 (2025)** | **2030세대 74.2% · 여성 89.1%** · 뷰티/건강/식품 관심 | 2030, 특히 여성이 **돈 걸고 습관 형성**에 가장 적극적 |
| **Setlog (2026)** | 학생~젊은 직장인 소그룹 (2~10명) · 카톡 단톡방 문화 | **친구 소그룹 카메라 공유** 문화 이미 정착 |

**교집합** → 친구 네트워크 보유 · 2030 · 운동 재시작 의지 · 소액 지출 가능 · 카톡/인스타 네이티브.

### 2.2 Primary Persona ⭐ "돌아가고 싶은 운동러 민지"

POC 모집 타겟. 팀의 **모든 설계 판단**의 기준점입니다.

| 항목 | 값 |
|---|---|
| 가칭 | 민지 |
| 나이 | 29세 |
| 직업 | IT 회사 3년차 기획자 |
| 거주 | 서울 강남 오피스텔 |
| 소득 | 연 5,000만원 · 월 가처분 약 150만원 |
| 운동 경력 | 대학 때 헬스 경험 · 취업 후 3년간 **PT 2~3회 끊고 흐지부지** 반복 |
| 스마트워치 | **없음** (Galaxy Watch 살까 1년째 고민 중) |
| 주 사용 앱 | 카톡 · 인스타 · 배민 · 올리브영 |
| 관심사 | 건강, 자기관리, 소비, 여행, 뷰티 |
| 친구 밀도 | 연락 유지 친구 5~8명 · 정기적으로 대화하는 단톡방 3개 |

**민지의 현재 문제 (Pain Points)**

- 혼자 헬스장 등록 → **3주 만에 중단**을 3년간 반복.
- 동네 러닝 시도 → 비 오면 스킵, 다음 날부터 재시작 못 함.
- 챌린저스 시도 경험 있음 → **"혼자 돈 걸어도 외로워서"** 이탈.
- 단톡방에서 "다음 주부터 운동하자" 매주 말만.

**민지의 JTBD (Job-to-be-Done, 해결하고 싶은 과업)**

> When: 새해 / 휴가 직후 / 체중계 충격받은 날
> Job: **"친한 친구들과 2~3주만이라도 딱 꾸준히"** 운동하고 싶다
> Outcome: 안 하면 돈 나가니까 어쩔 수 없이 하게 됨 + 친구 시선으로 도피 방지

**민지의 Pain ↔ 우리 기능 매칭**

| 민지의 Pain | 우리 앱 기능 |
|---|---|
| 혼자서는 의지 부족 | 친구 4명 그룹 서약서 (§3 MVP #1) |
| 기록 남기기 귀찮음 (**타이핑 자체가 부담**) | **원탭 키워드 → AI 운동일기** (§3 MVP #3) — 메모 타이핑 없이 버튼만으로 일기 완성 |
| 실패 기록 마주보기 무서움 | 원탭 응원 Kudos (§3 MVP #5) — 실패해도 친구가 격려 |
| 큰돈 걸긴 부담 | 참가비 1천~2만원 소액 (챌린저스 선례와 정합) |
| 앱 깔기/설치 귀찮음 | 웹/PWA 우선 (§9 기술 스택) |

### 2.3 Secondary Persona — "대학생 친구 4인방"

v1 확장 타겟. POC에서 민지 페르소나가 검증되면 적극 모집합니다.

| 항목 | 값 |
|---|---|
| 구성 | 대학교 2~4학년 동성 친구 4명 |
| 맥락 | 방학 · 시험 종료 직후 "우리 뭐 같이 해보자" 분위기 |
| 참가비 | 1회 1,000~3,000원 소액 선호 |
| 강점 | 시간 많음 · 인증 빈도 높음 · 인스타 공유 문화 익숙 |
| 약점 | **그룹 1개 실패 시 전원 이탈** (binary risk) → retention 설계 시 경계 |

### 2.4 Anti-Persona — "우리가 만들지 않는 사람들" ❌

범위를 명확히 하기 위해 **피해야 할 사용자**도 명시합니다. 팀이 *"이 사람도 고려해야 하나요?"* 라고 물을 때 **"아니오"**의 근거가 됩니다.

| 타입 | 왜 우리 앱이 맞지 않는가 |
|---|---|
| **하드코어 러너 / 사이클리스트** | 이미 Strava · 자동 GPS · 세그먼트 경쟁이 필요. 우리 수동 인증은 답답함. |
| **혼자 습관 만들고 싶은 사람** | 챌린저스가 이미 7년간 서비스. 우리는 친구 풀이 필수. |
| **중·고등학생** | 결제/소셜 법적 제약. 카톡 외 커뮤니케이션 파편화. |
| **40대+ 직장인** | 친구 밀도 ↓, 카톡 푸시 문화가 Gen Z와 다름. 테스트 타겟에서 제외. |
| **운동 완전 초보 (월 1회 미만)** | 주 2~3회 기준이 오히려 진입 장벽. POC 후 별도 "부드러운 모드" 필요. |

### 2.5 페르소나 → 테스트 그룹 모집 가이드

POC 2주차 테스트 그룹 확보 시 **이 체크리스트로 필터**합니다. "아무나 붙이면 POC 결과가 왜곡됩니다."

- [ ] 2030 직장인 또는 대학생 (민지형 비중 ≥ 60%)
- [ ] 이미 친한 친구 3~4명 단톡방이 존재 (= 그룹 수급이 단톡방 접수 수준으로 쉬움)
- [ ] 최근 1년 내 "운동 시작 → 중단" 경험 **1회 이상**
- [ ] 챌린저스/Strava 경험 없거나 이탈 경험 有 (비교 인터뷰 가치)
- [ ] 카톡/앱 푸시 수락 의사 有

**모집 채널 (권장 순위)**

1. 팀원의 **회사 동료 단톡방** (통제 쉬움, 신뢰도 높음)
2. **대학 동아리 OB** 단톡방 (2030 밀집, 친구 밀도 ↑)
3. **인스타 스토리 DM** (랜덤 노출 → 필터링 단계 필요)

> POC 성공 여부는 **모집된 그룹의 페르소나 적합도**에 크게 좌우됩니다. 모집 단계에서 **민지형 vs 대학생 4인방** 비중을 의도적으로 설계해야 결과 해석이 가능합니다.

## 3. 핵심 아이디어 (MVP Features)

| # | 기능 | 설명 |
|---|------|------|
| 1 | **그룹 서약서** | 그룹 생성 → 챌린지 조건 설정 → **전원 서명** → 시작 |
| 2 | **운동 인증** | 위치 + 운동시간 + 사진/키워드 + 헬스데이터 중 **2개 이상** 충족 시 인증 (POC 단계에서는 사진 + 키워드만) |
| 3 | **원탭 키워드 → AI 운동일기** | **타이핑 금지가 기본**. 운동 종류별 **랜덤 키워드 칩 6~9개**가 뜨고, 사용자는 **1~3개만 탭** → AI가 자동으로 3~5줄 일기 생성. "🎲 다시 뽑기"로 키워드 재추첨 가능. 직접 쓰고 싶은 경우만 secondary로 메모 입력 허용 (escape hatch) |
| 4 | **시작 알림** | 그룹원이 운동 시작하면 실시간 푸시 |
| 5 | **원탭 응원 (Kudos)** | 그룹원 인증에 이모지 1탭 반응 (Strava 2025 데이터: kudos 연 140억 개 · 댓글 대비 허들 압도적 낮음) |
| 6 | **외부 데이터 연동** | 스마트워치 / 운동앱 데이터 (POC 이후) |

> **#3의 설계 철학**: 민지 Pain의 본질은 *"AI가 써줘도 메모를 쳐야 한다"*였습니다. **인증의 모든 입력을 원탭화**(사진 + 운동 종류 탭 + 키워드 탭)하여, Kudos(#5)와 같은 "원탭 우선" 원칙을 **생성 단계에도 관통**시킵니다. 자유 메모는 완전 제거하지 않고 접힌 "직접 쓰고 싶어요" 링크로 두어, 저항형 사용자와 H4 가설 대조군을 동시에 확보합니다.

## 4. POC 범위 (2주 스프린트)

### 4.1 POC 목표 ⚓ 절대 흔들리면 안 됨

1. 그룹 기반 운동 인증이 **실제로 반복되는가**
2. **원탭 키워드 → AI 일기**가 기록 부담을 줄이고, 사용자가 그 결과물을 **"내 것"으로 수용하는가**
3. 벌금(예정 금액)이 **행동을 바꾸는가**
4. UI는 운동에 집중, 구조는 **모든 습관을 담을 수 있게**

### 4.2 성공 기준

| 지표 | 기준 |
|------|------|
| 1인당 주간 인증 횟수 | **평균 2회 이상** |
| 그룹 유지율 (서명 → 2주차까지) | **50% 이상** |
| 서명률 (초대 → 서명 완료) | **70% 이상** |

### 4.3 스코프

✅ **해야 할 것 (MVP)**

- 그룹 생성 / 초대 링크 / 참여
- 주간 챌린지(퀘스트) 생성
- 서약서 **전원 서명** → 시작
- 운동 인증 (사진 + **운동 종류 탭 + 키워드 칩 1~3개 탭**)
- **원탭 키워드 → AI 운동일기** 자동 생성 (자유 메모는 접힌 secondary)
- 주간 진행률 + **예정 벌금** 표시
- 최소 알림 (`누가 운동 시작` / `오늘 안 하면 실패`)
- **원탭 이모지 반응** (Kudos 차용 — engagement 허들 최저)

❌ **이번 스프린트에서 절대 하지 말 것**

- 카카오뱅크 등 **금융 연동**
- **스마트워치 / 헬스앱** 연동
- **위치 기반** 인증
- **멀티 챌린지 타입 UI** (공부 / 금주 등)
- **정교한 디자인** 작업

### 4.4 결정된 정책

- **Day 1부터 경쟁 유도** — "이번 주 꼴찌 벌금 9,000원" 메시지 없이는 서비스가 돌지 않습니다.
- **알림은 MVP 필수** — `누가 시작` / `오늘 안 하면 실패` 두 종류는 최소.
- **최적 그룹 크기 = 4명** (2명 재미없음 / 3~4명 최고 / 6명+ 방관자 발생).

## 5. 데이터 모델 (확장 가능한 설계)

### 5.1 설계 원칙

- **`ActionLog` 하나로 모든 행동**(운동 / 공부 / 습관)을 수용.
- 챌린지 타입(`Challenge.type`)과 인증 방식(`Verification.type`)으로 확장.
- POC에서는 `type = "fitness"` / `verification = "photo"`만 사용.

### 5.2 엔티티 초안

```text
User
Group

Challenge
  - title
  - type            // "fitness" (POC) → "study" / "habit" 등 확장
  - goal            // 횟수
  - duration
  - penalty

ChallengeParticipant
  - status          // pending / accepted

ActionLog
  - challengeId
  - userId
  - metadata        // JSON (운동 종류, 강도, 메모, 사진 URL 등)

Verification
  - type            // manual / photo / location
```

### 5.3 확장 시 UX 동일, 내용만 변경

```text
🔥 인기 챌린지 템플릿
- 주 3회 운동
- 매일 30분 공부
- 7일 금주
- 아침 7시 기상
```

## 7. POC 이후 백로그 (Next Candidates)

POC 검증 이후 우선순위 검토 대상입니다. 일부는 MVP에도 가볍게 편입 가능합니다.

### 7.1 그룹 퀘스트 보드 (2가지 모드)

Strava의 Group Challenges 4타입 중 우리 포지션에 맞는 **2가지**를 채택합니다.

**(A) 경쟁 모드** — 개인별 진행률 + 꼴찌 벌금 (POC 기본)

```text
JJ: 2/3   icemint: 3/3   DDuDDu: 1/3   shushu: 1/3
```

**(B) 협력 모드** — 그룹 공동 목표 (Strava Group Goal 차용, v1에서 도입)

```text
우리 그룹: 12회 중 9회 달성 (75%)
→ 모두 성공하면 전원 상금 + 면제권
```

> 같은 서약서에 두 모드를 **토글 옵션**으로 제공하면 "경쟁형 그룹"과 "협력형 그룹" 모두 수용 가능합니다. 매일 앱을 열 이유를 만듭니다.

### 7.2 실패 예고 시스템

벌금은 **실패 후 부과보다 실패 직전 압박**이 더 효과적입니다.

- "오늘 안 하면 이번 주 실패 확률 82%"
- "아직 1회 남았어요. 28시간 남음."

### 7.3 주간 운동 브이로그

한 주 종료 시 자동 리캡 생성 — 인증 사진 / AI 일기 / 총 운동 횟수 / 벌금 / MVP 멤버. → 인스타·카톡 **자연스러운 공유 포인트**.

### 7.4 벌금 + 보상 구조

벌금만 있으면 서비스가 무거워집니다. 보상과 함께 설계합니다.

- 실패 벌금은 **모임통장** 적립.
- 월말 성공률 1위에게 커피.
- 전체 성공 시 벌금 면제권.
- 모인 돈으로 회식 / 운동용품 구매.

#### 핵심 메커니즘 (관찰)

- **자동 GPS 기록**: Garmin/Apple Watch/폰 GPS 연동 → 수동 입력 거의 불필요
- **Segments**: 인기 구간의 개인 최고 기록 + KOM/QOM (King/Queen of the Mountain) 리더보드
- **Kudos**: 좋아요 1단계 축소판 — 댓글보다 가벼운 반응, **"I see you"** 의 디지털 표현
- **Photos**: 활동에 사진 첨부 — **있으면 kudos 3.1배** 데이터 검증됨
- **Clubs**: 느슨한 커뮤니티 (수백~수천 명)
- **Group Challenges**: tight한 그룹 경쟁 — **4가지 타입**
  1. Most Activity (누적 시간/거리/고도 리더보드)
  2. Fastest Effort (평균 페이스)
  3. Longest Single Activity (최장 단일 활동)
  4. **Group Goal** (리더보드 없이 공동 기여) ⭐
- **2025년 8월부터 Group Challenges는 유료만** 가능 (Premium 전환 압박 강화)
- **주간 목표가 연간 목표보다 압도적 인기** ← 우리 설계와 직결

#### 🎯 우리 설계를 데이터로 검증하는 Strava 2025 팩트

아이디에이션 단계에서 "감"으로 결정한 것들이 **실제로 맞다**는 증거가 여기 있습니다.

| 우리 결정 | Strava 데이터 근거 |
|---|---|
| **주간 챌린지** 구조 (§6 Week 1~2) | "주간 목표가 Strava 최인기 목표 기간 — 연간 목표는 2월에 이미 실패" |
| **사진 인증** 필수 (§4.3 MVP) | 사진 첨부 활동은 **kudos 3.1배** — engagement driver |
| **"운동"을 첫 use case로** | Gen Z는 운동을 데이팅/술 대체 소셜로 채택 — **러닝 클럽 3.5배 증가** |
| **친구 기반 그룹** (§4.4 4명) | Strava 본인도 tight Group Challenges에 수요 확인 → 2025 Premium으로 보호 |

#### 우리가 피해야 할 실수 (Strava가 보여준 교훈)

| 문제 | Strava 현상 | 우리 대응 방안 |
|---|---|---|
| **개인 중심 → 그룹은 부록** | 본질이 개인 기록, 친구는 관람자 | 우리는 **그룹이 1st class** · 개인 기록은 그룹 맥락 안에서만 |
| **하드코어 러너/사이클리스트 쏠림** | 초보자 진입 허들 (페이스/케이던스 같은 용어) | POC는 **"운동 = 버튼 한 번"** UI · 전문 용어 배제 |
| **GPS 의존 = 헬스/요가 2등 시민** | 헬스장/요가 기록은 저평가 | `ActionLog.metadata` 유연 · **운동 종류 평등** |
| **기능 비대화** | Premium 무엇이 무료/유료인지 혼란 | POC~v1은 **단일 플랜** · v2부터 Premium 도입 |
| **Follower 기반 Privacy** | "팔로워만/전체공개" 정책이 복잡 | 우리는 **그룹 내부로만 공개** — 정책 단순화 |

#### 포지셔닝 차이

| 축 | Strava | 우리 앱 |
|---|---|---|
| 관계 | 개인 + 느슨한 Follow/Club (수백~수천) | **친한 친구 4명 tight** |
| 대상 | 하드코어 러너/사이클리스트 | 일반 운동 초보 / 습관 관리 |
| 기록 방식 | **자동 GPS** (강점) | 수동 사진 + **원탭 키워드**(POC) → 워치 연동은 v2 |
| 동기 원천 | 기록 단축 · KOM · Kudos | **벌금 + 친구 시선 + 그룹 목표** |
| BM | Freemium ($180M+) | POC 무BM → 모임통장 → v2 수수료 → 장기 제휴 |
| 강점 | 네트워크 효과 · GPS 데이터 자산 | **관계 기반 결속** · 행동 강제 |

> **결론**: Strava는 우리에게 **"직접 경쟁 상대"가 아닌 "설계 검증용 레퍼런스"** 입니다.
> - 우리의 **주간 챌린지 · 사진 인증 · AI 일기 · 그룹 구조**는 Strava 2025 데이터로 전부 근거가 섰습니다.
> - 우리가 차용할 것은 UX 세부 3가지: **① Kudos 스타일 가벼운 반응 · ② Photo-first 피드 · ③ Group Goal 협력 모드**.
> - 우리가 경계할 것은 **"개인 중심 + 하드코어 집중 + 기능 비대화"** 의 함정.
> - 스마트워치/자동 인증 연동 시점(§4.3 Out of Scope → v2 편입)이 오면 Strava API / Runna(AI 코칭) 통합 방식을 다시 학습합니다.

## 용어집

- **Anti-persona (안티 페르소나)** — "우리가 의도적으로 대상 삼지 않는" 사용자 상. 범위를 명확히 해 요청/기능 편성을 방어하는 장치.
- **BM (Business Model, 비즈니스 모델)** — 서비스가 수익을 만드는 구조.
- **BeReal** — 정해진 시점에 알림이 울려 즉석 사진을 찍어 공유하는 SNS. "타이밍 사진" UX의 원형.
- **binary risk (이원적 리스크)** — 성공/실패가 "전부 아니면 전무"로 갈리는 리스크. 그룹 1개 실패 시 멤버 전원이 이탈하는 구조가 예.
- **CPA (Cost Per Action, 행동당 비용)** — 광고 과금 방식의 하나. 사용자가 특정 행동(설치·인증 등)을 완료했을 때 광고주가 지불.
- **DAU (Daily Active Users, 일간 활성 사용자)** — 하루 동안 제품을 사용한 고유 사용자 수.
- **Dogfood / dogfooding** — 팀 자체가 실제 사용자처럼 제품을 써 보는 것. POC Week 2의 주된 활동.
- **Escape hatch (비상구)** — 주 경로가 막혔을 때 빠져나갈 대안 경로. 여기서는 키워드 대신 "직접 메모 쓰기".
- **FOMO (Fear Of Missing Out, 소외 공포)** — 남들이 하는 것을 나만 놓친다는 불안. Setlog/BeReal의 주된 동력.
- **Freemium** — 기본은 무료, 고급 기능은 유료로 제공하는 과금 방식.
- **Gen Z** — 1990년대 중반~2010년대 초반 출생 세대. 2030세대와 겹침.
- **GPS (Global Positioning System)** — 위성 기반 위치 측정. Strava의 자동 기록 핵심.
- **IPO (Initial Public Offering, 기업공개)** — 회사가 주식을 거래소에 상장해 공개 거래하도록 만드는 것.
- **JTBD (Job-to-be-Done, 해결하고 싶은 과업)** — "사용자가 이 제품으로 해내고 싶은 일"을 상황·동기·결과 중심으로 기술하는 프레임워크.
- **KOM/QOM (King/Queen of the Mountain)** — Strava의 세그먼트 최고 기록자 타이틀.
- **Kudos (쿠도스)** — 칭찬·응원. Strava가 "가벼운 좋아요"로 정착시킨 UX.
- **Loss aversion (손실 회피)** — 이득보다 손실을 더 크게 느끼는 인간 심리. 벌금 설계의 근거.
- **MAU (Monthly Active Users, 월간 활성 사용자)** — 한 달 동안 제품을 사용한 고유 사용자 수.
- **MVP** — 여기서는 두 가지 의미가 섞여 등장하므로 주의:
  - **Minimum Viable Product (최소 기능 제품)** — §3, §4.3의 MVP Features.
  - **Most Valuable Player (최우수 참가자)** — §7.3의 "MVP 멤버".
- **North Star (북극성)** — 제품이 장기적으로 지향하는 방향. 기능 결정이 흔들릴 때 재조정 기준.
- **PO (Product Owner, 제품 책임자)** — 제품 방향·스코프·우선순위 결정자. `with-key`는 Ian.
- **POC (Proof of Concept, 개념 증명)** — 핵심 가설을 짧은 기간에 검증하기 위한 최소 구현. 2주.
- **PWA (Progressive Web App)** — 설치형 앱처럼 동작하는 웹앱. 오프라인·알림·홈 화면 아이콘 지원.
- **Retention (리텐션)** — 사용자가 일정 기간 후에도 남아 있는 비율. 제품 건강의 핵심 지표.
- **SAM (Serviceable Addressable Market)** — 전체 시장(TAM) 중 실제 진입 가능한 부분.
- **Segments (Strava Segments)** — 인기 러닝/사이클 구간에 대한 개인 최고 기록.
- **SOM (Serviceable Obtainable Market)** — SAM 중 단기간 내 확보 가능한 부분.
- **Strava** — 피트니스 소셜 네트워크. §8.3에서 상세 분석.
- **TAM (Total Addressable Market, 전체 시장)** — 제품이 이론적으로 닿을 수 있는 최대 시장.
- **Zero-sum (제로섬)** — 한쪽의 이득이 정확히 다른 쪽의 손실인 구조. 챌린저스의 "실패자→성공자 재분배".
