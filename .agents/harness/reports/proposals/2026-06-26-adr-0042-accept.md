## 대상

ADR-0042의 `**Status**` 를 `proposed` → `accepted` 로 승격한다. 승격이 박는 것은 **실행 기질 결정의 방향과 제약**이지 즉시 전환이 아니다(ADR-0042 Decision: "즉시 전환 아님. inline + worktree 가 POC 현행 기본 유지").

## Level 분류

**Level 2 — 제안만(PR + meta-eval + 사람)**.

근거: [UPDATE_POLICY](../../UPDATE_POLICY.md) §읽는 workflow — "권한 경계 자체 변경 = Level 2 + meta-eval 자기참조(AUTONOMY_EXPANDED 등)". ADR-0042 는 무인 루프의 실행 기질·권한 경계(step별 격리 헤드리스 세션 + multi-tool 자율)를 다루므로 권한 경계 변경에 해당한다. ADR 본문 line 7 이 이미 자신을 "Level 2 + AUTONOMY_EXPANDED"로 자기참조한다.

- **Level 1 아님**: 경로/스크립트명 단순 반영이 아니다.
- **Level 3 아님**: PRD goal·MVP scope·eval 수용기준(계약)을 바꾸지 않는다 — 실행 기질은 머시너리 거버넌스 결정이지 제품 계약이 아니다.

## meta-eval 결과

**weaken** — reason-code **AUTONOMY_EXPANDED**.

- **분류 근거(결정론)**: 채택 결정이 _기록상_ 하네스 자율 범위를 넓힌다 — 향후 step별 격리 헤드리스 세션 + multi-tool executor 차용 방향을 "승인된 방향"으로 박제한다. 자율 경계 확대 = AUTONOMY_EXPANDED([05-rn-harness-decisions](../../../../docs/migration/05-rn-harness-decisions.md) §6.1).
- **×3 침식 경보**: **해당 없음**. AUTONOMY_EXPANDED 의 **첫 실사용**(기존 출현은 enum 정의뿐 — config·plan·spec·05, 실제 meta-eval 결정 0건). 이후 같은 코드 재발 시 ×3 침식 추적의 1번째 카운트.
- **neutral 로 볼 여지(PO 판단 참고)**: 현행 mechanics 를 0 변경하고(inline+worktree 유지), 불변(D6 사람 게이트·meta-eval·회고 루프)을 명시 보존하며, 차용 시 3개 결함 수정(외부 oracle·worktree·allowlist)을 *강제*해 오히려 거버넌스를 조인다. **그럼에도 보수적으로 weaken 분류** — 자율 경계를 *승인*하는 결정은 자동 머지로 흘려보내지 않고 PO 사인을 거치게 하는 것이 게이트 설계 의도다(under-classify 하면 게이트가 무력화된다).

## risks

- 승격이 "방향 승인"으로 읽혀 후속 전환 spec 이 게이트(별도 spec + AUTONOMY_EXPANDED meta-eval + PO)를 건너뛸 오해 위험. → ADR 본문이 G1~G6 + 별도 게이트를 명시하므로 완화. 본 proposal 의 next-step 도 재확인.
- AUTONOMY_EXPANDED 첫 사용 — 이후 같은 코드 재발 시 ×3 체계적 침식 추적이 여기서 시작된다(추적 책임 명시).
