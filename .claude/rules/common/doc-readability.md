# 문서 가독성 · 주니어 친화 작성 규칙 (always-on)

**대상 독자**: 이 프로젝트의 **1년차 주니어 엔지니어**.
**적용 범위**: `docs/**`, 코드 주석, PR 본문, 커밋 메시지, 에이전트가 생성하는 모든 새 문서/수정.

## 왜

- 문서는 "필자가 쉽게 쓰기" 위한 것이 아니라 "처음 보는 사람이 따라 읽기" 위한 것이다.
- 시니어에게는 명확한 약어(RLS, RSC, VAPID)가 주니어에게는 진입 장벽이다.
- 진입 장벽이 높은 문서는 결국 아무도 안 읽고, 온보딩 비용이 사람에게 전가된다.

## 문장/단락

- **한 문장 = 한 개념**. 쉼표로 3개 이상 개념을 이어 붙이지 않는다.
- 한 단락은 3~5줄 이내를 목표로. 더 길어지면 bullet로 분해.
- 수동태·긴 명사구(~에 의한 ~화) 대신 능동태·짧은 동사.
- "그것", "이것", "해당 부분" 같은 애매한 지시어를 피하고 구체 대상을 다시 적는다.

## 전문 용어 다루기

1. **약어/영문 용어는 문서 안에서 처음 등장할 때** 괄호로 풀어쓴다.
   - 예: `RLS(Row Level Security, 행 단위 접근 제어)` · `RSC(React Server Component)` · `SoT(Source of Truth, 단일 출처)`
2. **도메인 관용어**(서약서, 키워드 풀, Quiet Hours 등)도 프로젝트 내 의미를 1줄로 풀어준다.
3. **문서 맨 아래에 `## 용어집` 섹션**을 두고 본문에서 푼 용어를 다시 모아 알파벳/가나다 순으로 정리한다.
   - 한 번 푼 용어라도 용어집에 중복 수록 OK. 독자는 중간부터 읽을 수도 있다.
4. 코드 식별자(`createServerClient`, `AnalyticsEvent`)는 용어집 대상이 아니다 — 코드 레벨 이름은 그대로 둔다.

## 외부 문서 참조

- "PRD §5.3" / "BE_SCHEMA §3" 같이 외부 섹션 번호만 던지지 않는다.
- 참조 옆에 **그 섹션이 무엇을 말하는지 한 줄 요약**을 덧붙인다.
  - 예: `PRD §5.3 (AI 일기 AC — 4.5초 타임아웃 · 키워드 커버리지 < 1 시 폴백)`

## 코드 블록

- 경로가 있는 코드 예제는 첫 줄에 주석으로 파일 경로를 적는다. 예: `// src/lib/ai/diary.ts`
- 길어지면 "...(중략)"으로 자르고, 주니어가 알아야 할 부분만 남긴다.
- 셸 명령은 한 줄에 한 개. `&&`로 이어 붙이지 않는다(에러 지점을 찾기 어렵다).

## "왜"를 함께 남기기

- 결정/제약/금지 사항에는 **반드시 이유 1줄**을 단다.
  - 나쁜 예: `SWR · React Query 도입 금지`
  - 좋은 예: `SWR · React Query 도입 금지 — POC 범위 초과이고, 쓰기는 Server Action으로 일원화되어 클라이언트 캐시 계층이 불필요하다.`

## 섹션 도입부

- 각 `##`/`###` 헤딩 바로 아래에 **그 섹션이 무엇을 다루는지 1~2줄**로 도입부를 둔다.
- 목차만 보고도 필요한 섹션으로 점프할 수 있도록.

## 금기 표현

- "당연히", "간단히", "쉽게", "누구나 알듯이" — 주니어가 자신을 탓하게 만든다.
- "우리가 알다시피" — 새로 온 사람은 모른다.
- "TBD만 잔뜩" — 비어있는 섹션은 `> _채울 예정: YYYY-MM-DD 담당자_` 형태로 **누가/언제**를 명시.

## 기존 문서 리라이팅 체크리스트

- [ ] 헤딩 바로 아래 1~2줄 도입부가 있는가
- [ ] 약어/영문 용어 첫 등장 시 풀어쓰기가 있는가
- [ ] 외부 문서 섹션 참조에 한 줄 요약이 붙어 있는가
- [ ] 결정/금지 항목에 "왜" 한 줄이 붙어 있는가
- [ ] 문서 맨 아래 `## 용어집`이 있는가 (3개 이상 전문 용어가 등장하는 문서라면 필수)
- [ ] "당연히/쉽게/누구나" 같은 금기 표현이 남아 있지 않은가

## 적용 우선순위

1. **`docs/`의 팀 공유 문서** — ARCHITECTURE, PRD, BE_SCHEMA, ONBOARDING, DECISIONS, VALIDATION, IDEATION, KICKOFF, CLAUDE
2. 코드 주석 (특히 `lib/ai`, `lib/keywords`, `middleware.ts`, `lib/supabase/*`)
3. PR 본문 / 커밋 메시지 (`.claude/commands/*.md` 템플릿에서 사용)

## 예외

- `.claude/drafts/` 아래 개인 드래프트 · KICKOFF 같은 로컬 전용 문서는 이 규칙을 강제하지 않는다(공유 문서로 승격할 때 전체 통과시킨다).
