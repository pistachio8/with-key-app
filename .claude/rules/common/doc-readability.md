# 문서 가독성 · 주니어 친화 작성 (always-on)

대상: 1년차 주니어 엔지니어. 적용: `docs/**`, 코드 주석, PR 본문, 커밋 메시지, 에이전트 생성 문서.

문서는 "처음 보는 사람이 따라 읽기" 위한 것. 약어/관용어가 진입 장벽이 되지 않도록 한다.

## 문장·단락

- 한 문장 = 한 개념. 쉼표로 3개 이상 개념을 잇지 않는다.
- 한 단락 3~5줄 이내. 길어지면 bullet로 분해.
- 능동태·짧은 동사 (수동태·긴 명사구 지양).
- "그것/이것/해당 부분" 같은 애매한 지시어 대신 구체 대상 명시.

## 전문 용어

1. 약어/영문 용어 첫 등장 시 괄호로 풀어쓴다.
   - `RLS(Row Level Security)` · `RSC(React Server Component)` · `SoT(Source of Truth)`
2. 도메인 관용어(서약서·키워드 풀·Quiet Hours 등)도 1줄 의미 추가.
3. 문서 맨 아래 `## 용어집` 섹션 — 본문에서 푼 용어를 가나다순으로 모음 (3개 이상 등장 시 필수).
4. 코드 식별자(`createServerClient`)는 용어집 대상 아님.

## 외부 문서 참조

섹션 번호만 던지지 않는다. 한 줄 요약을 덧붙인다.
- 예: `PRD §5.3 (AI 일기 AC — 4.5초 타임아웃, 키워드 커버리지 < 1 시 폴백)`

## 코드 블록

- 경로 있는 예제는 첫 줄에 파일 경로 주석. `// src/lib/ai/diary.ts`
- 길어지면 "...(중략)"으로 자르고 핵심만 남김.
- 셸 명령 한 줄에 한 개. `&&`로 잇지 않는다 (에러 지점 추적이 어렵다).

## "왜"를 함께 남긴다

결정/제약/금지에는 이유 1줄.
- 나쁜 예: `SWR · React Query 도입 금지`
- 좋은 예: `SWR · React Query 도입 금지 — POC 범위 초과, 쓰기는 Server Action 일원화로 클라이언트 캐시 불필요`

## 섹션 도입부

각 `##`/`###` 헤딩 바로 아래 1~2줄 도입부. 목차만 보고 점프 가능하도록.

## 금기 표현

- "당연히/간단히/쉽게/누구나 알듯이" — 주니어가 자신을 탓하게 만든다.
- "우리가 알다시피" — 새로 온 사람은 모른다.
- TBD 빈 섹션은 `> _채울 예정: YYYY-MM-DD 담당자_`로 누가/언제 명시.

## 리라이팅 체크리스트

- [ ] 헤딩 아래 1~2줄 도입부
- [ ] 약어 첫 등장 풀어쓰기
- [ ] 외부 참조에 한 줄 요약
- [ ] 결정/금지에 "왜"
- [ ] 3개 이상 용어 시 `## 용어집`
- [ ] 금기 표현 제거

## 적용 우선순위

1. `docs/` 팀 공유 문서 (ARCHITECTURE · PRD · BE_SCHEMA · ONBOARDING · DECISIONS · VALIDATION 등)
2. 코드 주석 (특히 `lib/ai` · `lib/keywords` · `middleware.ts` · `lib/supabase/*`)
3. PR 본문 / 커밋 메시지

예외: `.claude/drafts/`·KICKOFF 같은 로컬 전용은 강제하지 않음 (공유 문서로 승격 시 통과).
